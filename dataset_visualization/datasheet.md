# Evaluation Dataset Datasheet — Bbox Format

## Output Bbox Format (all JSONL files)

All output files use the same normalized corner-format bbox:

```json
"bbox": [x, y, w, h]
```

| Field | Description |
|-------|-------------|
| `x`   | Left edge of the bounding box (normalized 0–1) |
| `y`   | Top edge of the bounding box (normalized 0–1) |
| `w`   | Width of the bounding box (normalized 0–1) |
| `h`   | Height of the bounding box (normalized 0–1) |

- **Origin:** upper-left corner of the image is (0, 0).
- **Reference frame:** 512x512 center-cropped image (resize shortest side to 512, then center crop).
- **Clamping:** all values are clamped to [0, 1] after conversion.
- Values are rounded to 6 decimal places.

## Source Bbox Formats

### Hidden Objects (HO)

Raw format in `results.jsonl` from the verifier stage:

```json
"detected_bbox": {"x1": ..., "y1": ..., "x2": ..., "y2": ...}
```

- `x1, y1`: upper-left corner in **pixel coordinates** (512x512 image).
- `x2, y2`: lower-right corner in **pixel coordinates** (512x512 image).
- Already in the 512x512 center-crop space (no resize/crop transform needed).

**Conversion** (`convert_ho_bbox`):
```
x = x1 / 512,  y = y1 / 512,  w = (x2 - x1) / 512,  h = (y2 - y1) / 512
```

### OPA (Object Placement Assessment)

Raw format in `train_set_update.csv` / `test_set_update.csv`:

```json
"position": [x, y, w, h]
```

- `x, y`: upper-left corner in **original image pixel coordinates** (before any resize/crop).
- `w, h`: width and height in **original image pixel coordinates**.

**Conversion** (`convert_opa_bbox`):
1. Compute the resize scale: `scale = 512 / min(orig_w, orig_h)`.
2. Scale bbox to resized image: `x_scaled = x * scale`, etc.
3. Subtract center-crop offset: `x_crop = x_scaled - offset_x`.
4. Normalize to 0–1: `x_n = x_crop / 512`.
5. Clamp all values to [0, 1].

## Upstream: Bbox Generator + Verifier (Stage 1–3)

Bboxes are generated by `bbox_generator.py` and stored in `bboxes.json`:

```json
{"bbox_id": 0, "x1": 64, "y1": 128, "x2": 256, "y2": 320}
```

- `x1, y1`: upper-left corner in **pixel coordinates** (512x512).
- `x2, y2`: lower-right corner in **pixel coordinates** (512x512).

The verifier (stage 3) loads these, passes them to Grounded-SAM for IoU matching,
and writes `detected_bbox` in the same `{x1, y1, x2, y2}` pixel format into `results.jsonl`.

## Downstream Model Bbox Format (PlacementDETR)

PlacementDETR trains directly on the JSONL `"bbox"` field, so it uses the **same corner format** as the dataset:

```
[x, y, w, h]   (all normalized 0–1)
```

- The `bbox_head` outputs 4 Sigmoid values → naturally in [0, 1].
- GIoU loss interprets these as corner-format: `x2 = x + w`, `y2 = y + h`.
- **No center-format conversion** anywhere in training or inference.

## Summary

| Source | Raw Format | Raw Coords | Output Format |
|--------|-----------|------------|---------------|
| bboxes.json | `{x1, y1, x2, y2}` | pixels (512x512) | — |
| HO verifier | `{x1, y1, x2, y2}` | pixels (512x512) | `[x, y, w, h]` normalized |
| OPA | `[x, y, w, h]` | pixels (original) | `[x, y, w, h]` normalized |
| PlacementDETR | `[x, y, w, h]` | normalized (0–1) | `[x, y, w, h]` normalized |
